## Executive Summary

bmad_test는 자기주도 학습자를 위한 macOS 기반 학습 집중 및 진척 관리 앱이다. 단순히 포모도로 타이머를 제공하는 데 그치지 않고, 사용자가 주제별 학습 세션을 기록하고 주간 목표 시간을 설정하며, 실제 학습 진척과 목표 달성 상태를 명확하게 확인할 수 있도록 돕는다.

이 제품이 해결하려는 핵심 문제는 사용자가 매일 꾸준히 학습하거나 특정 활동을 지속하고 싶어도, 실제로 얼마나 해냈는지 체계적으로 추적하고 주간 단위의 지속성을 유지하기 어렵다는 점이다. 하루 이틀 계획이 어긋나더라도 주간 목표 관점에서 학습을 이어갈 수 있게 하여, 사용자가 장기적인 꾸준함을 유지하도록 지원하는 것이 핵심 가치다.

기존의 일반 포모도로 앱은 집중 시간 측정에는 유용하지만, 주제별 학습 흐름과 세부 카테고리별 기록, 목표 기반 진척 관리, 통계적 피드백이 부족하다. 반면 노션이나 수기 기록 방식은 유연하지만 기록 부담이 크고 즉시성 있는 피드백과 동기부여를 제공하기 어렵다.

bmad_test는 학습 주제와 태그 기반의 세션 기록, 주간 목표 시간 설정, 성과 통계, 데이터 export를 결합해 사용자가 "내가 이번 주에 얼마나 해냈는지", "어떤 주제에 시간을 쓰고 있는지", "목표 달성까지 얼마나 남았는지"를 한눈에 파악할 수 있도록 설계된다. 이를 통해 사용자는 성취감과 동기부여를 얻고, 자신의 학습 루틴을 더 지속 가능하게 관리할 수 있다.

## Core Vision

### Problem Statement

자기주도 학습자는 꾸준히 공부하거나 특정 활동을 지속하고 싶어 하지만, 실제 학습 시간을 주제별로 구조적으로 기록하고 주간 목표 달성 여부를 확인할 수 있는 도구가 부족해 지속성을 유지하기 어렵다. 단순히 오늘 했는지 여부가 아니라, 한 주 전체 기준으로 계획 대비 얼마나 실행했는지를 확인할 수 있어야 꾸준함을 유지할 수 있다.

### Problem Impact

이 문제가 해결되지 않으면 사용자는 공부를 하고도 얼마나 해냈는지 명확히 파악하지 못하고, 하루 이틀 계획이 어긋났을 때 쉽게 흐름이 끊긴다. 기록은 불완전해지고, 학습 주제별 투자 시간도 모호해지며, 결국 성취감과 동기부여가 약해진다. 장기적으로는 학습 루틴 유지가 어려워지고, 목표 달성 가능성도 낮아진다.

### Why Existing Solutions Fall Short

일반 포모도로 앱은 학습 시간 측정에는 적합하지만, 주제별 분류, 태그 기반 세부 카테고리 기록, 주간 목표 관리, 통계 분석 측면에서 한계가 있다. 반면 노션이나 수기 기록 방식은 사용자가 원하는 구조로 관리할 수 있지만 입력 부담이 크고, 실시간 진척 확인이나 동기부여를 위한 피드백 경험이 약하다. 공부 타이머 앱도 존재하지만, 꾸준함 유지와 목표 달성 추적이라는 측면에서 충분히 구조화된 경험을 제공하지 못하는 경우가 많다.

### Proposed Solution

bmad_test는 macOS에서 사용할 수 있는 학습 집중 앱으로, 기본 포모도로 사이클(예: 25분 학습, 5분 휴식)을 제공하면서 사용자가 학습 주제를 만들고, 각 주제 아래 태그나 세부 카테고리를 지정해 세션을 기록할 수 있도록 한다. 사용자는 주제별 주간 목표 시간을 설정하고, 실제 누적 학습 시간과 남은 목표 시간을 확인할 수 있다. 앱 내 통계에서는 전체 진행 시간뿐 아니라 주제별, 태그별 세부 분석을 제공하며, 일간/주간/월간/연간 단위의 JSON export도 가능하게 한다.

### Key Differentiators

- 단순 타이머가 아니라 주간 목표 달성을 중심으로 꾸준함을 설계한 학습 앱
- 주제와 태그 단위로 학습 세션을 세분화해 통계적으로 분석 가능한 구조
- 하루 실패 여부보다 주간 누적 진척을 강조해 지속성을 유지하도록 돕는 경험
- 사용자가 자신의 학습 성과를 명확히 확인하고 성취감과 동기부여를 얻을 수 있는 피드백 중심 설계
- macOS 환경에서 빠르고 집중감 있는 개인 학습 워크플로우에 최적화된 경험

## Target Users

### Primary Users

bmad_test의 주요 사용자는 자기주도적으로 공부 계획을 세우고 실행하는 학습자들이다. 대표적인 1차 사용자는 공무원 시험, 자격증 시험, 대학 전공 시험 등을 준비하는 수험형 학습자다.

이 사용자 페르소나는 예를 들어 "민준, 27세, 직장과 공부를 병행하는 자격증 준비생"으로 구체화할 수 있다. 민준은 매주 공부 목표를 세우지만, 실제로는 과목별로 얼마나 시간을 썼는지 정확히 기록하지 못한다. 어떤 날은 열심히 했다고 느끼지만, 주간 단위로 보면 목표 대비 부족한 경우가 많다. 반대로 하루 계획이 틀어지면 전체 루틴이 무너졌다고 느껴 쉽게 동기부여를 잃는다.

현재 이들은 일반 포모도로 앱, 노션, 종이 플래너, 메모 앱 등을 조합해 공부를 관리한다. 하지만 시간 측정과 기록, 목표 관리, 통계 확인이 분절되어 있어 한 흐름으로 이어지지 않는다. 이 때문에 기록 부담이 생기고, 실제 진척을 체계적으로 확인하기 어렵다.

이 사용자는 공부 시작 전에 오늘 진행할 주제와 세부 태그를 선택하고, 세션 종료 후 누적 시간과 주간 목표 대비 진척을 확인하며, 주말이나 하루 마감 시점에 통계를 보면서 스스로를 점검하는 흐름에서 가장 큰 가치를 얻는다. 이들에게 성공이란 "이번 주 수학 복습 목표 3시간 중 2시간 40분을 해냈다"처럼 자신의 노력이 구체적으로 보이고, 계획이 완벽하지 않아도 꾸준함을 유지할 수 있다고 느끼는 것이다.

### Secondary Users

2차 사용자로는 혼자 공부하는 대학생, 개발 공부를 하는 직장인, 독서나 글쓰기 같은 자기계발 활동을 꾸준히 관리하고 싶은 사용자도 고려할 수 있다. 이들은 반드시 시험 준비생은 아니지만, 특정 주제에 시간을 투자하고 장기적으로 루틴을 유지하고 싶다는 점에서 유사한 니즈를 가진다.

장기적으로는 학습 외에도 운동, 독서, 사이드 프로젝트, 습관 형성 같은 영역으로 확장될 수 있다. 다만 초기 제품에서는 자기주도 학습자를 중심으로 설계하고, 이후 인접 사용 사례로 확장하는 것이 더 적절하다.

### User Journey

사용자는 먼저 공부를 시작하기 전에 앱을 열고 오늘 진행할 학습 주제를 선택하거나 새로 만든다. 필요하다면 "복습", "문제풀이", "개념 정리" 같은 태그를 지정한 뒤 포모도로 세션을 시작한다.

세션이 끝나면 사용자는 해당 학습 시간이 자동으로 기록되고, 주제별 누적 시간과 주간 목표 대비 남은 시간이 업데이트되는 것을 확인한다. 짧은 휴식 후 다시 다음 세션을 이어가면서, 하루 동안 자신이 어떤 주제에 얼마만큼 집중했는지를 자연스럽게 축적한다.

사용자가 가치를 강하게 체감하는 순간은 단순히 타이머가 끝날 때가 아니라, 누적 기록이 쌓여 "내가 이번 주 목표에 얼마나 가까워졌는지"를 확인하는 시점이다. 특히 하루 이틀 계획이 어긋났더라도 주간 관점에서 진척이 보이면, 실패감 대신 다시 이어갈 수 있다는 동기부여를 얻는다.

장기적으로 이 앱은 사용자의 학습 루틴 속에서 공부 시작 전 계획 설정, 세션 중 집중, 세션 후 기록 확인, 주간 마감 시 통계 점검이라는 반복 패턴의 일부가 된다. 그렇게 함으로써 단순 타이머가 아니라 꾸준함을 유지하는 학습 운영 도구로 자리 잡는다.

## Success Metrics

bmad_test의 성공은 사용자가 단순히 타이머를 한 번 실행하는 것이 아니라, 자신의 학습 루틴을 주간 단위로 유지하고 실제 목표 달성에 가까워지고 있다고 느끼는지로 판단한다. 따라서 가장 중요한 성공 순간은 사용자가 주간 목표를 달성했거나, 목표에 얼마나 가까워졌는지를 명확히 확인하며 성취감과 동기부여를 얻는 순간이다.

이 제품이 사용자에게 가치를 제공하고 있다는 신호는 다음과 같은 행동에서 확인할 수 있다.

- 사용자가 반복적으로 앱을 열고 학습 세션을 시작한다.
- 사용자가 주제와 태그를 활용해 자신의 학습 시간을 구조적으로 기록한다.
- 사용자가 주간 목표를 설정하고, 남은 목표 시간을 확인하며 학습을 이어간다.
- 사용자가 통계 화면을 통해 자신의 진척을 점검한다.

초기 단계에서 특히 중요한 사용자 성공 지표는 다음과 같다.

- 주간 목표 달성률
- 주간 총 학습 시간
- 주제별 누적 학습 시간
- 사용자가 통계 화면을 확인하는 빈도

### Business Objectives

이 제품은 초기 기준으로 개인용 앱이므로, 매출이나 팀 단위 운영 지표보다는 사용자가 실제로 꾸준함을 유지하는 데 도움이 되는지에 초점을 둔다. 따라서 현재 단계의 핵심 목표는 사용자 한 명에게도 분명한 가치를 제공하는 제품 경험을 만드는 것이다.

초기 제품 목표는 다음과 같다.

- 사용자가 주제별로 학습 세션을 쉽게 기록할 수 있게 한다.
- 사용자가 주간 목표를 설정하고 진척을 확인할 수 있게 한다.
- 사용자가 통계와 export 기능을 통해 자신의 학습 데이터를 신뢰할 수 있게 한다.
- 사용자가 하루 실패 여부가 아니라 주간 누적 진척을 기준으로 학습을 계속 이어가도록 돕는다.

### Key Performance Indicators

초기 KPI는 다음과 같이 정의할 수 있다.

- 주간 목표 달성률: 설정한 주간 목표 시간 대비 실제 달성 비율
- 주간 총 학습 시간: 한 주 동안 누적된 전체 학습 시간
- 주제별 누적 학습 시간: 특정 주제별로 사용자가 투자한 시간
- 주간 세션 실행 횟수: 한 주 동안 시작된 포모도로 학습 세션 수
- 통계 화면 조회 빈도: 사용자가 자신의 진척을 확인하기 위해 통계 화면에 접근한 횟수

기간 기준이나 목표 수치는 아직 확정되지 않았으므로, 이후 PRD 단계에서 보다 구체적인 목표값과 측정 기준을 설정하는 것이 적절하다.

## MVP Scope

### Core Features

첫 버전의 MVP는 사용자의 핵심 문제인 "학습 루틴을 주간 단위로 유지하고 진척을 명확히 확인하는 것"을 해결하는 데 필요한 최소 기능에 집중한다. 반드시 포함되어야 할 핵심 기능은 다음과 같다.

- 포모도로 타이머: 기본 학습/휴식 사이클 설정 및 실행
- 주제 생성 및 관리: 사용자가 학습 주제를 만들고 선택할 수 있는 기능
- 세션 기록: 각 학습 세션의 시간과 연결된 주제를 자동으로 기록하는 기능
- 주간 목표 설정: 주제별 주간 학습 목표 시간을 설정하는 기능
- 기본 통계: 주간 목표 대비 진행률과 전체 학습 시간을 확인할 수 있는 화면

이 조합이 있어야 사용자는 단순히 시간을 재는 수준을 넘어, 실제로 무엇을 얼마나 했는지와 목표 대비 얼마나 진행했는지를 확인할 수 있다.

### Out of Scope for MVP

첫 버전에서는 핵심 가치 전달에 직접적이지 않거나 범위를 크게 확장시키는 기능은 제외한다. 우선 제외할 항목은 다음과 같다.

- JSON export
- 월간/연간 통계
- 태그별 상세 분석

태그 자체를 완전히 배제할지는 이후 PRD에서 조정할 수 있지만, 최소한 고도화된 태그 분석과 확장형 리포트는 MVP 범위 밖으로 두는 것이 적절하다.

### MVP Success Criteria

MVP의 성공 기준은 복잡한 시장 지표가 아니라, 실제 사용자가 이 앱을 일주일 동안 사용해 보며 학습 목표 추적에 실질적인 도움을 받는지에 있다. 따라서 초기 성공은 다음과 같이 판단할 수 있다.

- 사용자가 1주일 동안 실제 학습 루틴에 앱을 적용할 수 있다.
- 사용자가 주제별 학습 시간과 주간 목표 진척을 무리 없이 확인할 수 있다.
- 사용자가 "나는 이번 주에 얼마나 했는지"를 명확히 이해할 수 있다.
- 사용자가 단순 타이머보다 더 큰 성취감과 동기부여를 느낀다.

### Future Vision

장기 방향은 일단 macOS 개인용 앱에 집중한다. 즉, 초기 제품은 혼자 사용하는 학습 운영 도구로 완성도를 높이는 데 초점을 둔다. 이후 확장 가능성은 열어둘 수 있지만, 현재 단계에서는 iPhone 연동, 협업 기능, 대규모 플랫폼화보다는 macOS 환경에서 빠르고 안정적이며 개인 학습 루틴에 깊이 맞는 경험을 만드는 것이 우선이다.
