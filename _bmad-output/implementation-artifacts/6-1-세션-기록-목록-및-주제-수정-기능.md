# Story 6.1: 세션 기록 목록 및 주제 수정 기능

Status: done

## Story

As a 학습자,
I want 완료된 세션 기록의 주제를 수정할 수 있기를,
so that 잘못 선택한 주제가 통계를 왜곡하지 않고 기록을 신뢰할 수 있다.

## Acceptance Criteria

1. **Given** 통계 화면(`/stats`)에서 세션 기록 목록이 표시될 때 **When** 사용자가 기록 목록 섹션을 보면 **Then** 완료된 학습 세션 기록이 최신순으로 표시된다 (FR24)
2. **And** 각 기록 항목에는 주제명, 학습 시간(분), 날짜가 표시된다
3. **Given** 특정 기록의 수정 옵션을 선택하면 **When** Record Correction Item/Dialog이 표시될 때 **Then** 주제 필드는 수정 가능하고 다른 주제를 선택할 수 있다 (FR24, UX-DR6)
4. **And** 시간 필드는 수정 불가로 표시되며 "세션 시간은 실제 기록이므로 수정할 수 없습니다" 이유가 안내된다 (FR25)
5. **And** 수정 가능/불가 상태가 시각적·텍스트로 명확히 구분된다 (UX-DR6)
6. **And** 폼 필드에 명확한 레이블이 연결된다 (UX-DR14)
7. **Given** 주제를 변경하고 저장하면 **Then** 세션 기록의 `topic_id`가 DB에서 업데이트된다
8. **And** 업데이트 후 목록에 변경된 주제명이 즉시 반영된다

#### 1. `reassignSessionTopic()` — session-repository.ts 추가

```typescript
// src/domain/sessions/session-repository.ts — 새 함수 추가

import type { ReassignSessionTopicInput } from './session';
import { ReassignSessionTopicSchema } from './session-schema';

/**
 * 완료/중단된 세션의 주제를 변경한다.
 * - Zod 검증 → 세션 존재+완료/중단 확인 → 새 topicId 존재 확인 → UPDATE topic_id → 갱신된 row 반환
 */
export async function reassignSessionTopic(input: ReassignSessionTopicInput): Promise<Result<Session>> {
  try {
    const parsed = ReassignSessionTopicSchema.safeParse(input);
    if (!parsed.success) {
      const issue = parsed.error.issues[0];
      return err(ERROR_CODES.VALIDATION_ERROR, issue?.message ?? '입력값이 올바르지 않습니다');
    }
    const { sessionId, newTopicId } = parsed.data;

    // 1. 세션 존재 + 상태 확인
    const sessionRows = await select<SessionRow>('SELECT * FROM sessions WHERE id = $1', [sessionId]);
    if (sessionRows.length === 0) {
      return err(ERROR_CODES.NOT_FOUND, '세션을 찾을 수 없습니다');
    }
    const session = toSession(sessionRows[0]);
    if (session.status !== 'completed' && session.status !== 'interrupted') {
      return err(ERROR_CODES.SESSION_STATE_CONFLICT, '완료되거나 중단된 세션만 주제를 변경할 수 있습니다');
    }

    // 2. 새 topicId 존재 확인
    const topicRows = await select<{ id: string }>('SELECT id FROM topics WHERE id = $1', [newTopicId]);
    if (topicRows.length === 0) {
      return err(ERROR_CODES.NOT_FOUND, '주제를 찾을 수 없습니다');
    }

    // 3. UPDATE topic_id
    const now = Date.now();
    await execute(
      'UPDATE sessions SET topic_id = $1, updated_at_ms = $2 WHERE id = $3',
      [newTopicId, now, sessionId],
    );

    // 4. 갱신된 row 반환
    const rows = await select<SessionRow>('SELECT * FROM sessions WHERE id = $1', [sessionId]);
    if (rows.length === 0) {
      return err(ERROR_CODES.PERSISTENCE_ERROR, '세션 갱신 후 조회에 실패했습니다');
    }
    return ok(toSession(rows[0]));
  } catch (error) {
    return err(
      ERROR_CODES.PERSISTENCE_ERROR,
      `세션 주제 변경 중 오류가 발생했습니다: ${error instanceof Error ? error.message : String(error)}`,
    );
  }
}
```

#### 6. record-correction-service.ts

```typescript
// src/features/records/record-correction-service.ts
import {
  findSessionsByDateRange,
  reassignSessionTopic,
} from '../../platform/browser/session-adapter';
import { findAllTopics } from '../../platform/browser/topic-adapter';
import type { Result } from '../../shared/lib/result';
import type { Session, ReassignSessionTopicInput } from '../../domain/sessions/session';
import { ok, err } from '../../shared/lib/result';
import { ERROR_CODES } from '../../shared/lib/errors';
import type { Topic } from '../../domain/topics/topic';

export interface SessionRecordItem {
  session: Session;
  topicName: string;
}

/**
 * 최근 세션 기록 목록을 로드한다.
 * - 학습 세션(study)만, 완료 또는 중단 상태만 표시
 * - 주제명을 조인
 */
export async function loadSessionRecords(): Promise<Result<SessionRecordItem[]>> {
  // 최근 30일 범위 조회 (충분한 기록 확보)
  const now = Date.now();
  const thirtyDaysAgo = now - 30 * 24 * 60 * 60 * 1000;

  const [sessionsResult, topicsResult] = await Promise.all([
    findSessionsByDateRange(thirtyDaysAgo, now),
    findAllTopics(),
  ]);

  if (!sessionsResult.ok) return err(sessionsResult.code, sessionsResult.message);
  if (!topicsResult.ok) return err(topicsResult.code, topicsResult.message);

  const topicMap = new Map<string, string>();
  for (const topic of topicsResult.data) {
    topicMap.set(topic.id, topic.name);
  }

  const records: SessionRecordItem[] = sessionsResult.data
    .filter((s) => s.phaseType === 'study' && (s.status === 'completed' || s.status === 'interrupted'))
    .map((session) => ({
      session,
      topicName: topicMap.get(session.topicId) ?? '삭제된 주제',
    }));

  return ok(records);
}

/**
 * 세션 기록의 주제를 변경한다.
 */
export async function reassignRecordTopic(
  sessionId: string,
  newTopicId: string,
): Promise<Result<Session>> {
  return reassignSessionTopic({ sessionId, newTopicId });
}
```

#### 7. useRecordCorrection 훅

```typescript
// src/features/records/hooks/useRecordCorrection.ts
import { useState, useEffect, useCallback } from 'react';
import { loadSessionRecords, reassignRecordTopic, type SessionRecordItem } from '../record-correction-service';
import { useSessionStore } from '../../session/state/sessionStore';

export function useRecordCorrection() {
  const lastCompletedAtMs = useSessionStore((state) => state.lastCompletedAtMs);
  const [records, setRecords] = useState<SessionRecordItem[]>([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const fetchRecords = useCallback(async () => {
    setIsLoading(true);
    setError(null);
    const result = await loadSessionRecords();
    if (result.ok) {
      setRecords(result.data);
    } else {
      setError(result.message);
    }
    setIsLoading(false);
  }, []);

  useEffect(() => {
    fetchRecords();
  }, [fetchRecords, lastCompletedAtMs]);

  const reassignTopic = useCallback(async (sessionId: string, newTopicId: string) => {
    const result = await reassignRecordTopic(sessionId, newTopicId);
    if (result.ok) {
      await fetchRecords(); // 목록 갱신
    }
    return result;
  }, [fetchRecords]);

  return { records, isLoading, error, refetch: fetchRecords, reassignTopic };
}
```

### Anti-Patterns to Avoid

- ❌ `sessions` 테이블에 새 컬럼 추가 — 불필요. 기존 `topic_id` 컬럼을 UPDATE하면 됨
- ❌ REST API 엔드포인트 생성 — MVP에 네트워크 API 없음 (architecture.md)
- ❌ 통계 재계산 로직을 이 스토리에 포함 — Story 6.2 범위. 이 스토리는 DB UPDATE + UI만 담당
- ❌ 시간(duration) 수정 기능 구현 — FR25에 의해 명확히 금지됨
- ❌ `RecordCorrectionItem`을 별도 route(`/records`)로 만들기 — 기록 목록은 `/stats` 내 섹션으로 배치
- ❌ 세션 삭제 기능 — 에픽 6 범위 밖, 데이터 무결성 원칙에 반함
- ❌ `appStore` 같은 새 글로벌 store 생성 — 기존 패턴 유지 (Zustand는 feature별 소규모)
- ❌ `RecordList`에서 직접 SQL/adapter 호출 — service → adapter → repository 계층 따를 것
- ❌ `reassignSessionTopic`에서 running 상태 세션 수정 허용 — completed/interrupted만 허용
- ❌ topic-adapter의 `findAllTopics`를 사용하지 않고 직접 SQL로 주제 조회 — adapter 경계 준수

### 데이터 흐름 시퀀스

```
    → topic-adapter.findAllTopics() → topics 데이터
    → 조인하여 SessionRecordItem[] 생성 → RecordList 렌더링

주제 수정:
  사용자 "수정" 클릭 → RecordCorrectionDialog 열림
    → 활성 주제 목록 표시 (useTopics 또는 직접 topic-adapter)
    → 새 주제 선택 → "저장" 클릭
    → reassignTopic(sessionId, newTopicId)
      → session-adapter.reassignSessionTopic()
        → session-repository.reassignSessionTopic()
          → UPDATE sessions SET topic_id = $1 WHERE id = $2
      → 성공 시 fetchRecords() 재호출 → 목록 갱신
      → 통계는 Story 6.2에서 재계산 (이 스토리에서는 DB만 업데이트)
```
