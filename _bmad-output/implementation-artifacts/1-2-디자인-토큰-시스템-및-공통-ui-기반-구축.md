### 색상 토큰 설계 가이드

UX 문서 기준으로 아래 방향을 따라 색상 토큰을 정의한다:

- **Primary 색상**: 저채도 블루 또는 틸 계열 — 차분하고 신뢰감 있는 느낌
- **배경**: 밝고 부드러운 뉴트럴 톤
- **성공**: 과도한 축하색 대신 조용한 성취감 (예: 저채도 그린)
- **경고/오류**: 공격적 빨강 최소화. 목표 미달/세션 중단을 실패처럼 보이게 하지 않는 것이 원칙
- **Destructive**: 명확하되 최소 범위에만 빨강 적용
- Semantic 토큰 (용도 기반): `--color-primary`, `--color-secondary`, `--color-success`, `--color-warning`, `--color-error`, `--color-bg-*`, `--color-text-*`, `--color-border-*`

### 타이포그래피 토큰 설계 가이드

- 시스템 폰트 스택 사용 (추가 폰트 로드 없음)
- 계층 구분:
  - Display/h1: 타이머 큰 숫자, Summary Card 핵심 수치
  - h2/h3: 섹션 제목
  - body: 일반 텍스트
  - caption/small: 보조 설명
- 숫자/시간 정보는 시각적 우선순위 최상위
- `font-size`, `line-height`, `font-weight`를 토큰으로 정의

### Spacing 토큰 설계 가이드

8px 기반 체계:
```
--spacing-1: 4px    (0.5x)
--spacing-2: 8px    (1x)
--spacing-3: 12px   (1.5x)
--spacing-4: 16px   (2x)
--spacing-5: 20px   (2.5x)
--spacing-6: 24px   (3x)
--spacing-8: 32px   (4x)
--spacing-10: 40px  (5x)
--spacing-12: 48px  (6x)
--spacing-16: 64px  (8x)
```

### Button 컴포넌트 설계 가이드

```typescript
// src/shared/ui/Button/Button.tsx
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'text' | 'destructive';
  size?: 'small' | 'medium' | 'large';
  isLoading?: boolean;
  children: React.ReactNode;
}
```

**Variant 스타일 원칙:**
- `primary`: 가장 높은 대비, 명확한 채움형. 한 화면에 하나만 사용
- `secondary`: 경계선 또는 약한 채움
- `text`: 배경 없음, 텍스트만
- `destructive`: 최소 범위 빨강, 위험 행동용

**상태 스타일 원칙:**
- `hover`: 색상 밝기/채도 미세 변화
- `focus-visible`: 포커스 링 2px solid offset, 충분한 대비
- `disabled`: opacity 감소 + cursor: not-allowed
- `aria-disabled="true"` 사용하여 접근성 확보

### CSS Modules 패턴

```css
/* Button.module.css */
.button { /* 기본 스타일 */ }
.primary { /* primary variant */ }
.secondary { /* secondary variant */ }
.text { /* text variant */ }
.destructive { /* destructive variant */ }
.small { /* small size */ }
.medium { /* medium size */ }
.large { /* large size */ }
.disabled { /* disabled state */ }
```

```typescript
// Button.tsx에서 사용
import styles from './Button.module.css';
import clsx from 'clsx'; // 또는 간단한 자체 유틸
```

> **주의**: `clsx` 또는 `classnames` 라이브러리를 설치하지 않고, 간단한 className 결합 유틸을 `src/shared/lib/cn.ts`에 직접 구현해도 되고, 또는 template literal로 처리해도 된다. 외부 의존성 추가는 최소화한다.

### Anti-Patterns to Avoid

- ❌ Tailwind, utility-first CSS 프레임워크 설치
- ❌ styled-components, emotion 등 CSS-in-JS 라이브러리 사용
- ❌ 전역 CSS 파일에서 컴포넌트 스타일 작성 (CSS Modules 사용)
- ❌ 하드코딩된 색상/spacing/font-size 값 (반드시 CSS 변수 참조)
- ❌ `outline: none` — 포커스 표시를 제거하는 스타일
- ❌ 색상만으로 상태(enabled/disabled 등)를 전달
- ❌ 과도한 외부 의존성 추가 (classnames, clsx 등 — 필요 시 자체 유틸)
