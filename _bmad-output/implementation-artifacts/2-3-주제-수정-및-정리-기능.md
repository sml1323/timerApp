# Story 2.3: 주제 수정 및 정리 기능

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 학습자,
I want 기존 주제의 이름을 수정하거나 사용하지 않는 주제를 정리할 수 있기를,
so that 주제 목록을 깔끔하게 유지할 수 있다.

## Acceptance Criteria

1. **Given** 주제 목록에서 특정 주제를 선택했을 때 **When** 수정 옵션을 선택하면 **Then** 주제 이름을 인라인 또는 다이얼로그로 수정할 수 있다 (FR11)
2. **And** 수정 후 목록에 즉시 반영된다
3. **And** 빈 이름으로 수정 시 검증 오류가 표시된다
4. **Given** 더 이상 사용하지 않는 주제가 있을 때 **When** 정리(아카이브) 옵션을 선택하면 **Then** 해당 주제가 주제 선택 목록에서 숨겨진다 (FR12)
5. **And** 기존 세션 기록과의 연결은 유지된다
6. **And** 데이터는 삭제되지 않고 아카이브 상태로 보존된다

### ⚠️ 핵심: 신규 코드 최소화 — 이미 구현된 것 재사용

Story 2.1에서 이미 완성된 repository 함수:
- `updateTopic(id: string, input: UpdateTopicInput): Promise<Result<Topic>>` — Zod 검증, 존재 확인, 중복 이름 검사, UPDATE 모두 포함
- `archiveTopic(id: string): Promise<Result<Topic>>` — 존재 확인, 이미 아카이브 여부 확인, UPDATE 포함
- `findAllTopics(false)` — 활성 주제만 조회 (아카이브된 주제 자동 제외)

→ **이 스토리에서는 domain 계층 코드를 전혀 수정하지 않는다.** 변경은 `features/topics/` 내에서만 발생한다.

### topic-service.ts 확장

```typescript
// src/features/topics/topic-service.ts — 기존 코드에 추가
import { updateTopic, archiveTopic } from '../../domain/topics/topic-repository';
import type { UpdateTopicInput } from '../../domain/topics/topic';

/** 주제 이름 수정 */
export async function editTopic(id: string, input: UpdateTopicInput): Promise<Result<Topic>> {
  return updateTopic(id, input);
}

/** 주제 아카이브 (정리) */
export async function removeTopic(id: string): Promise<Result<Topic>> {
  return archiveTopic(id);
}
```

### useTopics.ts 확장

```typescript
// src/features/topics/hooks/useTopics.ts — 기존 코드에 추가
import { editTopic, removeTopic } from '../topic-service';

// 기존 return에 추가:
const updateExistingTopic = useCallback(async (id: string, name: string) => {
  const result = await editTopic(id, { name });
  if (result.ok) {
    await fetchTopics();
  }
  return result;
}, [fetchTopics]);

const archiveExistingTopic = useCallback(async (id: string) => {
  const result = await removeTopic(id);
  if (result.ok) {
    await fetchTopics();  // 아카이브 후 목록 갱신 → 해당 주제 자동 제거
  }
  return result;
}, [fetchTopics]);

return { topics, isLoading, error, createNewTopic, updateExistingTopic, archiveExistingTopic, refetch: fetchTopics };
```

```tsx
//   <input id={`edit-topic-${topic.id}`} value={editName} ... />
//   {editError && <span role="alert">{editError}</span>}

//   4. !result.ok → result.message를 editError에 표시 (중복 이름 등)
```

### 아카이브 확인 다이얼로그 구현

현재 `src/shared/ui/Dialog/` 컴포넌트가 존재하지 않으므로, TopicCard 내부에서 간단한 확인 UI를 구현한다:

```tsx
// 방법 1 (권장): TopicCard 내부 인라인 확인 영역
// showArchiveConfirm === true 일 때:
<div className={styles.archiveConfirm} role="alertdialog" aria-label="주제 정리 확인">
  <p>'{topic.name}'을(를) 정리하시겠습니까?</p>
  <p className={styles.archiveHint}>주제 선택 목록에서 숨겨지지만, 기존 학습 기록은 유지됩니다.</p>
  <div className={styles.archiveActions}>
    <Button variant="text" onClick={cancelArchive}>취소</Button>
    <Button variant="destructive" onClick={confirmArchive} isLoading={isArchiving}>정리</Button>
  </div>
</div>

// confirmArchive:
// 1. setIsArchiving(true)
// 2. onArchive(topic.id) 호출
// 3. result.ok → 목록에서 자동 제거 (useTopics refetch)
// 4. !result.ok → 에러 메시지 표시
```

> **왜 삭제가 아니라 아카이브인가:** FR12에 따라 주제 데이터는 삭제되지 않고 `is_archived = 1`로 설정된다. `findAllTopics(false)`가 활성 주제만 반환하므로, 아카이브 후 목록을 새로고침하면 해당 주제가 자연스럽게 목록에서 사라진다. Epic 6의 기록 정정 기능에서 아카이브된 주제도 과거 세션 기록의 주제로 참조될 수 있으므로 데이터 보존이 필수.

### 스타일링 가이드

- 카드 액션 영역: 카드 우측에 수정/정리 버튼 배치
- 수정 모드: 전체 카드 영역이 수정 폼으로 전환, `--color-border-focus` 강조
- 에러 텍스트: `--color-error`, `--font-size-sm`
- 아카이브 확인 영역: 카드 아래 확장 또는 오버레이, 부드러운 배경색
- 정리 버튼: `Button` 컴포넌트 `variant="destructive"` 사용
- 수정 버튼: `Button` 컴포넌트 `variant="text"` 또는 `variant="secondary"` 사용
- 저장 버튼: `Button` 컴포넌트 `variant="primary"` + `isLoading` prop
- 간격: `--spacing-*` 토큰 사용
- 전환 효과: 수정 모드 진입/종료 시 `--transition-fast`

### 접근성 요구사항

- 수정 모드 진입 시 input에 자동 포커스 (`useEffect` + `ref.focus()`)
- 수정 모드의 `<input>`에 `<label>` 연결 (`htmlFor`/`id`)
- 검증 에러는 `role="alert"` 사용
- 아카이브 확인 영역은 `role="alertdialog"` + `aria-label`
- 저장 진행 중 버튼 `aria-busy="true"` + `disabled`
- 키보드: Enter 저장, Escape 취소
- 정리 버튼은 destructive variant로 시각적 경고
- Tab 키로 수정/정리 버튼 접근 가능

### Anti-Patterns to Avoid

- ❌ 컴포넌트에서 `topic-repository.ts` 직접 import — 반드시 `topic-service.ts` 경유
- ❌ 컴포넌트에서 `sql-client.ts` 또는 Tauri 플러그인 직접 호출
- ❌ `domain/topics/` 파일 수정 — 이 스토리는 features 레이어만 변경
- ❌ `findAllTopics(true)` 호출 — 아카이브된 주제는 목록에 표시하지 않음
- ❌ 주제를 실제 삭제 (`DELETE`) — 반드시 아카이브 (`is_archived = 1`) 처리
- ❌ `inline style` 사용 — 반드시 CSS Modules
- ❌ `className` 문자열 직접 연결 — `cn()` 유틸리티 사용
- ❌ `<label>` 없이 `<input>` 단독 사용
- ❌ 에러 메시지를 `alert()` 또는 `console.log`로만 표시 — 인라인 UI 필수
- ❌ Tailwind CSS 또는 styled-components 사용 — CSS Modules 전용
- ❌ 공통 Button 재구현 — `src/shared/ui/Button` 재사용
- ❌ Zustand store 생성 — useState + refetch 패턴 유지
- ❌ Topic Quick Select Panel 구현 — Story 2.4 범위
- ❌ 아카이브 복원(unarchive) 기능 구현 — MVP 범위 외
- ❌ 확인 다이얼로그 없이 바로 아카이브 — 사용자 확인 필수
