## Story

As a 학습자,
I want 학습 주제 데이터가 안정적으로 저장되고 관리되기를,
so that 앱을 재실행해도 내 주제 목록이 보존된다.

## Acceptance Criteria

1. **Given** `topics` 테이블이 DB에 존재할 때 **When** 주제 도메인 모듈을 구현하면 **Then** `src/domain/topics/topic.ts`에 Topic 타입이 정의된다 (id, name, isArchived, createdAtMs, updatedAtMs)
2. **And** `topic-schema.ts`에 Zod 검증 스키마가 정의된다
3. **And** `topic-repository.ts`에 CRUD 함수가 구현된다 (create, findAll, findById, update, archive)
4. **And** `topic-mappers.ts`에 DB row(`snake_case`) ↔ TS 객체(`camelCase`) 매핑이 구현된다
5. **And** 모든 repository 함수가 discriminated result 객체(`{ ok, data }` / `{ ok, false, code, message }`)를 반환한다
6. **And** 빈 이름이나 중복 이름에 대한 검증이 동작한다

### topic-mappers.ts 설계 가이드

```typescript
// src/domain/topics/topic-mappers.ts
import type { Topic } from './topic';

/** DB row 타입 — snake_case */
export interface TopicRow {
  id: string;
  name: string;
  is_archived: number; // SQLite: 0 | 1
  created_at_ms: number;
  updated_at_ms: number;
}

/** DB row → TS 객체 변환 */
export function toTopic(row: TopicRow): Topic {
  return {
    id: row.id,
    name: row.name,
    isArchived: row.is_archived === 1,
    createdAtMs: row.created_at_ms,
    updatedAtMs: row.updated_at_ms,
  };
}
```

> **핵심 규칙:** `is_archived`(INTEGER 0/1) ↔ `isArchived`(boolean) 변환은 반드시 mapper에서만 수행. 컴포넌트나 repository SQL에서 직접 변환 금지.

### topic-repository.ts 설계 가이드

**함수 시그니처:**

```typescript
import { select, execute } from '../../platform/tauri/sql-client';
import { ok, err, type Result } from '../../shared/lib/result';
import { ERROR_CODES } from '../../shared/lib/errors';
import type { Topic, CreateTopicInput, UpdateTopicInput } from './topic';
import { type TopicRow, toTopic } from './topic-mappers';
import { CreateTopicSchema, UpdateTopicSchema } from './topic-schema';
```

**create 구현 패턴:**
1. Zod로 입력 검증 → 실패 시 `err(VALIDATION_ERROR, ...)`
2. 중복 이름 확인 — `SELECT id FROM topics WHERE name = $1 AND is_archived = 0`
3. 중복 발견 시 `err(VALIDATION_ERROR, '이미 존재하는 주제 이름입니다')`
4. `crypto.randomUUID()`로 id 생성
5. `Date.now()`로 `created_at_ms`, `updated_at_ms` 설정
6. `INSERT INTO topics (id, name, is_archived, created_at_ms, updated_at_ms) VALUES ($1, $2, $3, $4, $5)`
7. 삽입 후 `SELECT`로 조회하여 `toTopic(row)` 반환 → `ok(topic)`
8. DB 오류 발생 시 `err(PERSISTENCE_ERROR, ...)`

**findAll 구현 패턴:**
1. `includeArchived` 파라미터 기본값 `false`
2. `false`일 때: `SELECT * FROM topics WHERE is_archived = 0 ORDER BY created_at_ms DESC`
3. `true`일 때: `SELECT * FROM topics ORDER BY created_at_ms DESC`
4. rows 매핑 → `ok(topics)`

**findById 구현 패턴:**
1. `SELECT * FROM topics WHERE id = $1`
2. 결과 없으면 `err(NOT_FOUND, '주제를 찾을 수 없습니다')`
3. 결과 있으면 `ok(toTopic(row))`

**update 구현 패턴:**
1. Zod로 입력 검증
2. `findById`로 존재 확인
3. 중복 이름 확인 — 자신 제외: `WHERE name = $1 AND is_archived = 0 AND id != $2`
4. `UPDATE topics SET name = $1, updated_at_ms = $2 WHERE id = $3`
5. 갱신된 row 조회 후 `ok(topic)`

**archive 구현 패턴:**
1. `findById`로 존재 확인
2. 이미 아카이브됨 확인 → 에러 또는 무시
3. `UPDATE topics SET is_archived = 1, updated_at_ms = $1 WHERE id = $2`
4. 갱신된 row 조회 후 `ok(topic)`

**try/catch 패턴:**
모든 repository 함수는 전체 로직을 `try/catch`로 감싸고, catch에서는 `err(PERSISTENCE_ERROR, ...)` 반환. 절대 예외를 밖으로 throw 하지 않는다.

### Anti-Patterns to Avoid

- ❌ `sql-client.ts`의 `select`/`execute` 대신 `@tauri-apps/plugin-sql`의 `Database` 직접 import
- ❌ DB 컬럼명을 camelCase로 작성 (반드시 `snake_case`)
- ❌ Topic 타입에서 `is_archived: number` 사용 (반드시 `isArchived: boolean`으로 변환)
- ❌ mapper 밖에서 DB row ↔ TS 객체 변환 수행
- ❌ repository 함수에서 예외를 throw (반드시 `Result` 반환)
- ❌ `ok()`/`err()` 대신 직접 `{ ok: true, data }` 객체 생성 — 헬퍼 함수 사용
- ❌ UUID 생성에 외부 라이브러리 사용 — `crypto.randomUUID()` 사용
- ❌ 이 스토리에서 UI 컴포넌트, 훅, Zustand store 구현 (Story 2.2 범위)
- ❌ `topic-service.ts` 구현 — features 레이어는 이후 스토리
- ❌ `sessions`, `weekly_goals` 관련 코드 작성 — Epic 3, 4 범위
- ❌ 중복 이름 검사 시 아카이브된 주제 포함 — 활성 주제(`is_archived = 0`)만 검사
- ❌ `autoincrement` 또는 `INTEGER` PK — `TEXT` UUID 사용
- ❌ Zod 스키마에서 `z.string().nonempty()` 사용 — Zod 4에서는 `z.string().min(1)` 사용

### Previous Story Intelligence (Story 1.4)

**Story 1.4에서 확립된 패턴 (반드시 따를 것):**
- `sql-client.ts`의 `select<T>(query, bindValues)` / `execute(query, bindValues)` 사용

- `ERROR_CODES` 상수 — `src/shared/lib/errors.ts`

**`lastInsertId` 이슈:** execute() 반환 타입에서 `lastInsertId`는 optional (`number | undefined`). TEXT PK를 사용하므로 이 값은 무의미, 사용하지 않을 것.
