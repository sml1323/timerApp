### 아키텍처 패턴 — 반드시 따를 것

#### 1. SQL 집계 쿼리 패턴 (기존 `session-statistics.ts` 참고)

기존 `getWeeklyStudyMinutesByTopic`가 확립한 패턴을 동일하게 따른다:

```typescript
// 패턴 참고: src/domain/sessions/session-statistics.ts
import { select } from '../../platform/tauri/sql-client';
import { ok, err, type Result } from '../../shared/lib/result';
import { ERROR_CODES } from '../../shared/lib/errors';

export async function getTodayStudySummary(todayStartMs: number): Promise<Result<TodayStudySummary>> {
  try {
    const todayEndMs = todayStartMs + 24 * 60 * 60 * 1000;
    const rows = await select<StudySummaryRow>(
      `SELECT COALESCE(SUM(planned_duration_sec), 0) AS total_seconds,
              COUNT(*) AS session_count
       FROM sessions
       WHERE phase_type = 'study'
         AND status = 'completed'
         AND started_at_ms >= $1
         AND started_at_ms < $2`,
      [todayStartMs, todayEndMs],
    );
    return ok(toTodayStudySummary(rows[0]));
  } catch (error) {
    return err(
      ERROR_CODES.PERSISTENCE_ERROR,
      `오늘 통계 조회 중 오류: ${error instanceof Error ? error.message : String(error)}`,
    );
  }
}
```

#### 2. `getStudyByTopic` JOIN 패턴

```sql
SELECT s.topic_id, t.name AS topic_name,
       COALESCE(SUM(s.planned_duration_sec), 0) AS total_seconds,
       COUNT(*) AS session_count
FROM sessions s
JOIN topics t ON s.topic_id = t.id
WHERE s.phase_type = 'study'
  AND s.status = 'completed'
  AND t.is_archived = 0
GROUP BY s.topic_id, t.name
ORDER BY total_seconds DESC
```

⚠️ **아카이브 주제 처리**: `is_archived = 0`으로 활성 주제만 포함. 아카이브된 주제의 세션은 통계에서 제외하지 않으려면 이 필터를 제거해야 하나, Epic 5 AC에서는 "주제별 누적 학습 시간"으로만 명시되어 있으므로 **활성 주제만 표시**한다. 통계 완전성이 요구되면 별도 Story에서 처리.

#### 3. 매퍼 패턴 (기존 `session-mappers.ts` 참고)

```typescript
// src/domain/statistics/statistics-mappers.ts
import type { TodayStudySummary, WeeklyStudySummary, TopicStudySummary } from './statistics';

export interface StudySummaryRow {
  total_seconds: number;
  session_count: number;
}

export interface TopicStudySummaryRow {
  topic_id: string;
  topic_name: string;
  total_seconds: number;
  session_count: number;
}

const DEFAULT_SUMMARY = { totalMinutes: 0, sessionCount: 0 };

export function toTodayStudySummary(row: StudySummaryRow | undefined): TodayStudySummary {
  if (!row) return DEFAULT_SUMMARY;
  return {
    totalMinutes: Math.round(row.total_seconds / 60),
    sessionCount: row.session_count,
  };
}

export function toWeeklyStudySummary(row: StudySummaryRow | undefined): WeeklyStudySummary {
  if (!row) return DEFAULT_SUMMARY;
  return {
    totalMinutes: Math.round(row.total_seconds / 60),
    sessionCount: row.session_count,
  };
}

export function toTopicStudySummary(row: TopicStudySummaryRow): TopicStudySummary {
  return {
    topicId: row.topic_id,
    topicName: row.topic_name,
    totalMinutes: Math.round(row.total_seconds / 60),
    sessionCount: row.session_count,
  };
}
```

#### 4. Adapter 패턴 (기존 `session-adapter.ts` 참고)

```typescript
// src/platform/browser/statistics-adapter.ts
import { isTauriRuntime } from '../runtime/runtime-detect';
import type { TodayStudySummary, WeeklyStudySummary, TopicStudySummary } from '../../domain/statistics/statistics';
import type { Result } from '../../shared/lib/result';

export interface StatisticsRepositoryAdapter {
  getTodayStudySummary(todayStartMs: number): Promise<Result<TodayStudySummary>>;
  getWeeklyStudySummary(weekStartAtMs: number): Promise<Result<WeeklyStudySummary>>;
  getStudyByTopic(): Promise<Result<TopicStudySummary[]>>;
}

let adapter: StatisticsRepositoryAdapter | null = null;

async function getAdapter(): Promise<StatisticsRepositoryAdapter> {
  if (adapter) return adapter;

  if (isTauriRuntime()) {
    const mod = await import('../../domain/statistics/statistics-repository');
    adapter = mod;
  } else {
    const mod = await import('./in-memory-statistics-adapter');
    adapter = mod;
  }

  return adapter;
}

export async function getTodayStudySummary(todayStartMs: number): Promise<Result<TodayStudySummary>> {
  return (await getAdapter()).getTodayStudySummary(todayStartMs);
}

export async function getWeeklyStudySummary(weekStartAtMs: number): Promise<Result<WeeklyStudySummary>> {
  return (await getAdapter()).getWeeklyStudySummary(weekStartAtMs);
}

export async function getStudyByTopic(): Promise<Result<TopicStudySummary[]>> {
  return (await getAdapter()).getStudyByTopic();
}
```

#### 5. In-Memory Adapter 패턴

`in-memory-session-adapter.ts`의 `store` 배열을 직접 import해서 필터링:

```typescript
// src/platform/browser/in-memory-statistics-adapter.ts
// 핵심: in-memory-session-adapter.ts의 store를 export하도록 수정 OR
//       getSessionStore() 접근자 함수를 추가해 store에 접근한다.
```

⚠️ **중요**: `in-memory-session-adapter.ts`에서 `store`는 현재 `const store: Session[] = []`로 모듈 스코프에 선언되어 있으나 **export되지 않았다**. in-memory statistics adapter가 이 store에 접근하려면:
- 옵션 A: `store` export 추가 → `export const store: Session[] = []`
- 옵션 B: `getSessionStore(): Session[]` 접근자 함수 추가
- **옵션 B 권장** — 읽기 전용 접근 의도를 명확히 하고, 향후 store 구조 변경에 대응 가능

```typescript
// in-memory-session-adapter.ts에 추가
export function getSessionStore(): readonly Session[] {
  return store;
}
```

/**
 * 오늘 자정(UTC 기준)의 epoch ms를 반환한다.

export function getTodayStartAtMs(date: Date = new Date()): number {

}

### Anti-Patterns to Avoid

- ❌ `sessions` 테이블 외에 별도 `statistics` 테이블 생성 — sessions가 단일 사실원천, 매번 파생 계산
- ❌ `src/domain/sessions/session-statistics.ts`에 오늘/주간 집계 추가 — 그 파일은 주간 주제별 집계 전용(Story 4.3). 통계 도메인은 `src/domain/statistics/`에 분리
- ❌ 전체 세션을 TS로 가져와 클라이언트에서 집계 — SQL `GROUP BY`/`SUM`으로 처리
- ❌ `interrupted` 또는 `running` 세션을 통계에 포함 — `status = 'completed'`만
- ❌ `break` 세션을 학습 통계에 포함 — `phase_type = 'study'`만
- ❌ `endedAtMs - startedAtMs`로 실 학습 시간 계산 — `plannedDurationSec` 사용 (포모도로 완료 = 계획 시간)
- ❌ 매퍼에서 `snake_case` 필드를 그대로 export — repository/mapper 경계에서만 변환
- ❌ UI 컴포넌트 생성 — 이 스토리 범위는 도메인/데이터 레이어만
- ❌ Zustand store 변경 — 통계 상태는 Story 5.2/5.3 훅에서 로컬 관리
- ❌ `in-memory-session-adapter.ts`의 `store` 배열을 직접 노출 — `getSessionStore()` 접근자 사용
- ❌ `features/stats/statistics-service.ts` 구현 — 서비스 레이어는 Story 5.2/5.3에서
