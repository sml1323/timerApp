# Story 3.3: 세션 완료 및 자동 기록 저장

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 학습자,
I want 세션 완료 시 기록이 자동 저장되고 전진 결과를 확인할 수 있기를,
so that 이번 주 목표를 향해 실제로 전진하고 있다는 성취감을 얻는다.

## Acceptance Criteria

1. **Given** 포모도로 타이머가 0에 도달했을 때 **When** 세션이 완료되면 **Then** 세션 기록이 SQLite에 자동 저장된다 (FR6, NFR3)
2. **And** Session Outcome Panel이 표시된다 (UX-DR5 — success variant)
3. **And** 완료된 시간, 주제명이 결과에 표시된다
4. **And** `speak.svg` 캐릭터가 짧은 성취 피드백 문구와 함께 표시된다 (UX-DR4)
5. **And** "다음 세션 시작" 또는 "통계 보기" CTA가 제공된다
6. **And** 데이터는 의도하지 않은 손실 없이 저장된다 (NFR7)
7. **And** macOS 알림으로 세션 완료가 통지된다 (FR38, 선택적)

### ⚠️ 핵심: 기존 세션 완료 로직은 이미 동작 중

Story 3.2에서 **세션 완료 DB 저장은 이미 구현**되어 있다:
- `useSessionClock`의 `onComplete` 콜백 → `handleComplete()` → `completeStudySession(sessionId)` → DB 반영 + sessionStore `endSession()`
- 현재 `SessionPage.tsx`에서 `sessionPhase === 'completed'`일 때 간단한 텍스트("세션이 완료되었습니다") + 홈으로 돌아가기 버튼만 표시
- **이 스토리에서는 이 placeholder를 SessionOutcomePanel로 교체**하는 것이 핵심

따라서 **세션 완료 -> DB 저장 흐름은 건드리지 않고**, UI 결과 화면만 교체한다.

### ⚠️ sessionStore 확장 주의사항

현재 `endSession()` 액션:
```typescript
endSession: () => set({ sessionPhase: 'completed' }),
```

이것을 완료된 세션 데이터를 보관하도록 확장해야 한다:
```typescript
// 수정 방향
completedSession: null as Session | null,

endSession: (session: Session) =>
  set({ sessionPhase: 'completed', completedSession: session }),
```

**주의**: `endSession` 시그니처 변경이 `session-service.ts`의 호출부에도 반영되어야 한다:
```typescript
// session-service.ts 수정
if (result.ok) {
  useSessionStore.getState().endSession(result.data);
}
```

`initialState`에 `completedSession: null` 추가, `reset()`에서 자동 초기화됨.

### ⚠️ SessionOutcomePanel 구현 가이드

```typescript
// src/features/session/components/SessionOutcomePanel.tsx
import { CharacterStatePanel } from './CharacterStatePanel';
import { Button } from '../../../shared/ui/Button/Button';
import styles from './SessionOutcomePanel.module.css';

interface SessionOutcomePanelProps {
  variant: 'success' | 'recovery';
  topicName: string;
  durationSec: number;
  onNextSession: () => void;
  onViewStats: () => void;
  onGoHome: () => void;
}

export function SessionOutcomePanel({
  variant,
  topicName,
  durationSec,
  onNextSession,
  onViewStats,
  onGoHome,
}: SessionOutcomePanelProps) {
  const minutes = Math.floor(durationSec / 60);
  const seconds = durationSec % 60;
  const durationText = seconds > 0
    ? `${minutes}분 ${seconds}초`
    : `${minutes}분`;

  // success variant만 이 스토리에서 구현
  // recovery variant는 Story 3.5에서 구현
  const feedbackMessage = '수고했어요! 한 걸음 전진했습니다.';

  return (
    <div className={styles.panel}>
      <CharacterStatePanel state="speak" message={feedbackMessage} />
      <div className={styles.summary}>
        <p className={styles.topicName}>{topicName}</p>
        <p className={styles.duration}>{durationText} 학습 완료</p>
      </div>
      <div className={styles.actions}>
        <Button variant="primary" onClick={onNextSession}>
          다음 세션 시작
        </Button>
        <Button variant="secondary" onClick={onViewStats}>
          통계 보기
        </Button>
        <Button variant="text" onClick={onGoHome}>
          홈으로
        </Button>
      </div>
    </div>
  );
}
```

**UX 핵심**:
- 성취 피드백은 과도한 축하가 아닌 **조용하지만 분명한 만족감** (UX 감정 원칙)
- "얼마나 전진했는지"가 중심 (UX-DR5)
- 다음 행동 CTA는 홈으로만 보내지 않고 직접 선택지 제공 (Journey Pattern)

function computeActualDuration(): number {
  const completed = useSessionStore.getState().completedSession;
  if (completed?.endedAtMs && completed.startedAtMs) {

  return activeSession?.plannedDurationSec ?? 0;

### ⚠️ 알림(Notification) 구현 가이드

```typescript
// src/platform/tauri/notification-client.ts
import {
  isPermissionGranted,
  requestPermission,
  sendNotification,
} from '@tauri-apps/plugin-notification';

export async function sendSessionNotification(
  title: string,
  body: string,
): Promise<void> {
  try {
    let permissionGranted = await isPermissionGranted();
    if (!permissionGranted) {
      const permission = await requestPermission();
      permissionGranted = permission === 'granted';
    }
    if (permissionGranted) {
      sendNotification({ title, body });
    }
  } catch {
    // 알림 실패는 무시 — 핵심 기능 차단 금지
    console.warn('알림 발송 실패');
  }
}
```

**핵심 규칙**:
- 알림은 **선택적 보조 기능** — 실패해도 세션 완료 흐름에 영향 없음
- Tauri Notification 플러그인 미설치 시에도 앱이 정상 동작하도록 try/catch 필수
- 알림은 DB 저장 성공 **후**에만 발송 (Architecture: "Notification side effects occur after successful session state transitions")

### ⚠️ Tauri Notification 플러그인 설치 필요 여부

현재 `src-tauri/Cargo.toml`에 notification 플러그인이 없을 수 있다. 필요 시:

```bash
cd src-tauri

```

`src-tauri/src/lib.rs`에 플러그인 등록:
```rust

```

`package.json` 또는 npm:
```bash
npm install @tauri-apps/plugin-notification
```

`src-tauri/capabilities/default.json`에 권한 추가:
```json
"permissions": [
  "notification:default"
]
```

### Anti-Patterns to Avoid

- ❌ `session-repository.ts` 수정 — `completeSession`은 이미 Story 3.1에서 구현 완료
- ❌ `useSessionClock.ts` 수정 — 타이머 로직 변경 없음, `onComplete` 콜백 유지
- ❌ session 완료 DB 저장 중복 구현 — `completeStudySession`이 이미 처리
- ❌ recovery variant 구현 — Story 3.5 범위
- ❌ 휴식 세션 타이머/전환 — Story 3.4 범위
- ❌ 통계 계산/갱신 — Epic 5 범위
- ❌ `loading.svg`를 완료 상태에 사용 — 완료 시에는 반드시 `speak.svg` (UX-DR4)
- ❌ 과도한 축하 애니메이션/효과음 — 조용하지만 분명한 만족감 (UX 감정 원칙)
- ❌ Tailwind, styled-components, inline styles — CSS Modules + CSS variables만
- ❌ `console.info` placeholder 남기기 — 실제 동작하는 결과 화면으로 교체
- ❌ `features/dashboard/`, `features/stats/` 파일 생성/수정 — Epic 5 범위
- ❌ `topic-repository.ts` 수정 — 이 스토리와 무관
- ❌ 알림 실패 시 에러 UI 표시 — 알림은 보조 기능, 실패해도 무시
