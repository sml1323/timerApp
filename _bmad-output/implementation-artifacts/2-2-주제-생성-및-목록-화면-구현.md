## Story

As a 학습자,
I want 학습 주제를 생성하고 전체 주제 목록을 확인할 수 있기를,
so that 공부할 과목을 체계적으로 관리할 수 있다.

## Acceptance Criteria

1. **Given** 주제 관리 화면(`/topics`)에 접근했을 때 **When** 주제가 하나도 없으면 **Then** `default.svg` 캐릭터와 함께 "첫 학습 주제를 만들어보세요" 안내가 표시된다 (UX-DR8)
2. **And** 주제 생성 폼이 제공된다
3. **Given** 주제 생성 폼에 이름을 입력했을 때 **When** 저장 버튼을 누르면 **Then** 새 주제가 SQLite에 저장되고 목록에 즉시 나타난다 (FR9)
4. **And** 빈 이름 입력 시 인라인 검증 오류가 표시된다 (UX-DR14)
5. **And** 레이블이 폼 필드에 명확히 연결된다
6. **Given** 주제가 존재할 때 **When** 주제 목록 화면을 보면 **Then** 모든 활성 주제가 리스트로 표시된다 (FR10)
7. **And** 앱 재실행 후에도 목록이 보존된다 (FR34)

### topic-service.ts 설계 가이드

```typescript
// src/features/topics/topic-service.ts
import { createTopic, findAllTopics } from '../../domain/topics/topic-repository';
import type { Topic, CreateTopicInput } from '../../domain/topics/topic';
import type { Result } from '../../shared/lib/result';

/** 활성 주제 목록 조회 */
export async function loadTopics(): Promise<Result<Topic[]>> {
  return findAllTopics(false);
}

/** 새 주제 생성 */
export async function addTopic(input: CreateTopicInput): Promise<Result<Topic>> {
  return createTopic(input);
}
```

> **service 레이어의 역할:** 현재는 단순 위임이지만, 향후 Story 2.4(Topic Quick Select)에서 최근 선택 주제 표시 로직을 추가하거나, 통계 연동 시 오케스트레이션 로직이 들어갈 예정. 직접 repository를 컴포넌트에서 호출하지 않는 아키텍처 원칙을 따른다.

### useTopics.ts 설계 가이드

```typescript
// src/features/topics/hooks/useTopics.ts
import { useState, useEffect, useCallback } from 'react';
import type { Topic } from '../../../domain/topics/topic';
import { loadTopics, addTopic } from '../topic-service';

export function useTopics() {
  const [topics, setTopics] = useState<Topic[]>([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const fetchTopics = useCallback(async () => {
    setIsLoading(true);
    setError(null);
    const result = await loadTopics();
    if (result.ok) {
      setTopics(result.data);
    } else {
      setError(result.message);
    }
    setIsLoading(false);
  }, []);

  useEffect(() => {
    fetchTopics();
  }, [fetchTopics]);

  const createNewTopic = useCallback(async (name: string) => {
    const result = await addTopic({ name });
    if (result.ok) {
      // 생성 성공 시 목록 새로고침
      await fetchTopics();
    }
    return result;
  }, [fetchTopics]);

  return { topics, isLoading, error, createNewTopic, refetch: fetchTopics };
}
```

> **Zustand 미사용 이유:** Architecture 문서에 따르면 Zustand는 active session/UI 상태용. 주제 목록은 DB에서 읽어오는 persisted truth이므로 로컬 state + refetch 패턴이 적합. Story 2.4 전까지는 Zustand store 불필요.

### Anti-Patterns to Avoid

- ❌ 컴포넌트에서 `topic-repository.ts` 직접 import — 반드시 `topic-service.ts` 경유
- ❌ 컴포넌트에서 `sql-client.ts` 또는 Tauri 플러그인 직접 호출
- ❌ `findAllTopics(true)` 호출 (아카이브된 주제 포함) — Story 2.2는 활성 주제만 표시
- ❌ Zustand store 생성 — 주제 목록은 DB가 사실원천, 이 스토리에서는 useState + refetch 패턴 사용
- ❌ 주제 수정/아카이브 UI 구현 — Story 2.3 범위
- ❌ Topic Quick Select Panel 구현 — Story 2.4 범위
- ❌ `inline style` 사용 — 반드시 CSS Modules
- ❌ `className` 문자열 직접 연결 — `cn()` 유틸리티 사용
- ❌ 폼 필드에 `<label>` 없이 `<input>` 단독 사용
- ❌ 에러 메시지를 `alert()` 또는 `console.log`로만 표시 — 반드시 인라인 UI
- ❌ Tailwind CSS 또는 styled-components 사용 — CSS Modules 전용
- ❌ 캐릭터 SVG에 alt 텍스트 의존 — `aria-hidden="true"` + 별도 텍스트 동반 (UX-DR4)
- ❌ 공통 Button 재구현 — `src/shared/ui/Button` 재사용

### 접근성 요구사항

- 모든 폼 필드에 `<label>` 연결 (`htmlFor`/`id`)
- 검증 에러는 `role="alert"` 또는 `aria-live="polite"` 사용
- 저장 버튼 로딩 시 `aria-busy="true"`
- 키보드로 폼 전체 탐색 가능 (Tab 순서)
- 포커스 링 표시 (`--focus-ring-*` 토큰 활용)
- 주제 리스트는 의미론적 `<ul>/<li>` 또는 적절한 `role` 사용
- 빈 상태의 캐릭터 SVG는 장식이므로 `aria-hidden="true"`, 안내 텍스트가 의미 전달

### 스타일링 가이드

- 배경: `--color-bg-primary` 또는 `--color-bg-secondary`
- 텍스트: `--color-text-primary`, `--color-text-secondary`
- 카드: `--radius-md`, `--shadow-sm`, `--color-bg-elevated`
- 인풋: `--color-border-default`, focus 시 `--color-border-focus`
- 에러 텍스트: `--color-error`
- 간격: `--spacing-*` 토큰 사용 (4/8/12/16/24/32px)
- 폰트: `--font-family-base`, `--font-size-base`/`--font-size-sm`/`--font-size-lg`
- 전환 효과: `--transition-fast`, `--transition-normal`
