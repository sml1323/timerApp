## Story

As a 학습자,
I want 기록 수정 후 통계가 일관되게 재계산되기를,
so that 기록 정정 후에도 통계를 신뢰할 수 있다.

## Acceptance Criteria

1. **Given** 세션 기록의 주제가 변경된 후 **When** 저장 버튼을 누르면 **Then** 기록이 업데이트되고 관련 통계(오늘 누적, 주간 누적, 주제별 통계)가 자동 재계산된다 (FR26)
2. **And** `speak.svg` 캐릭터가 정정 완료를 안내하는 짧은 피드백 메시지를 표시한다 (UX-DR6, UX-DR4)
3. **And** 업데이트된 기록과 재계산된 통계 결과를 바로 확인할 수 있다
4. **And** 세션 결과가 일관되게 유지된다 (FR27) — 통계는 `sessions` 테이블 기준으로 재계산 (단일 사실원천)
5. **And** 피드백 메시지는 일정 시간(3초) 후 자동으로 사라지거나 사용자가 닫을 수 있다

## Dev Notes

### 핵심 이해: 이 스토리의 범위

이 스토리의 핵심은 **Story 6.1에서 구현된 `reassignSessionTopic` DB UPDATE 이후, 통계가 자동으로 일관되게 재계산되는지 검증하고, `speak.svg` 정정 완료 피드백 UI를 추가하는 것**이다.

아키텍처 문서에 명시된 **"Stats are recomputed from persisted records after write completion"** 원칙에 따라, 별도의 재계산 로직은 필요하지 않다. 통계는 `sessions` 테이블에서 매번 파생 계산되므로, `topic_id`가 UPDATE된 후 통계를 다시 조회하면 자동으로 올바른 결과가 나온다.

**따라서 이 스토리에서 새로운 통계 계산 로직을 작성하지 않는다.** 핵심 작업은:
1. 기존 흐름(`reassignTopic → refetchStats`)이 올바르게 동작하는지 검증
2. `speak.svg` 정정 완료 피드백 UI 컴포넌트 신규 생성
3. 테스트를 통한 데이터 일관성 검증


**결론: 재계산 흐름은 이미 완성되어 있다.** 이 스토리는 이를 검증하고 피드백 UI를 추가하는 것이 핵심이다.

### CorrectionFeedback 컴포넌트 상세 설계

```typescript
// src/features/records/components/CorrectionFeedback.tsx

import speakCharacter from '../../../assets/characters/speak.svg';
import styles from './CorrectionFeedback.module.css';
import { useEffect, useRef } from 'react';

interface CorrectionFeedbackProps {
  isVisible: boolean;
  onDismiss: () => void;
}

export function CorrectionFeedback({ isVisible, onDismiss }: CorrectionFeedbackProps) {
  const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

  useEffect(() => {
    if (isVisible) {
      // 3초 후 자동 닫기
      timerRef.current = setTimeout(onDismiss, 3000);
    }
    return () => {
      if (timerRef.current) clearTimeout(timerRef.current);
    };
  }, [isVisible, onDismiss]);

  if (!isVisible) return null;

  return (
    <div
      className={styles.feedback}
      role="status"
      aria-live="polite"
      onClick={onDismiss}
    >
      <img
        src={speakCharacter}
        alt=""
        aria-hidden="true"
        className={styles.character}
      />
      <p className={styles.message}>기록이 수정되었습니다. 통계에 반영되었어요.</p>
    </div>
  );
}
```

### Anti-Patterns to Avoid

- ❌ 별도의 `stats.recomputed` 이벤트 버스 구현 — MVP에 이벤트 버스 불필요 (architecture.md: "Avoid a generic event bus in MVP")
- ❌ 통계 재계산을 위한 별도 함수/서비스 생성 — 통계는 이미 sessions 테이블에서 파생 계산됨, `refetchStats()`만으로 충분
- ❌ `RecordCorrectionDialog` 안에서 직접 통계 refetch — 관심사 분리 위반: StatsRoute가 통합 책임
- ❌ Toast/Snackbar 공유 컴포넌트 신규 생성 — 이 스토리는 Record Correction 전용 피드백만 필요, 범용 토스트는 과도
- ❌ `sessionStore`에 새 상태 추가 — 피드백 표시는 StatsRoute 로컬 state로 충분
- ❌ 캐릭터에 speak.svg를 주요 정보 전달자로 사용 — 보조 역할만 (UX-DR4)
- ❌ `tokens.css`에 새 디자인 토큰 추가 — 기존 토큰으로 충분히 스타일링 가능

### 데이터 흐름 시퀀스

```

  사용자 "저장" 클릭 → RecordCorrectionDialog.handleSubmit()
    → StatsRoute.handleSave(sessionId, newTopicId)
      → useRecordCorrection.reassignTopic()
        → reassignRecordTopic() → session-adapter → session-repository
          → UPDATE sessions SET topic_id = newTopicId
        → 성공 시 fetchRecords() → 기록 목록 갱신
      → 성공 시 refetchStats() → useStatistics.fetchData()
        → loadStatsPageData() → 3개 쿼리 병렬 실행
          → getTodayStudySummary(todayStartMs) — sessions 재집계
          → getWeeklyStudySummary(weekStartMs) — sessions 재집계
          → getStudyByTopic() — sessions 재집계
        → 통계 UI 자동 갱신 (React state update → re-render)
      → 성공 시 setShowFeedback(true)
        → CorrectionFeedback 표시 (speak.svg + 완료 메시지)
        → 3초 후 자동 사라짐

핵심: topic_id가 UPDATE된 후 sessions 테이블에서 다시 SELECT하면
      자동으로 올바른 통계가 나온다. 별도 재계산 로직 불필요.
```
