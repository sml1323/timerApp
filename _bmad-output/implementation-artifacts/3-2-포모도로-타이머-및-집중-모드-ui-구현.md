### ⚠️ 타이머 구현 핵심 — timestamp 기반 (절대 interval 카운터 아님)

```typescript
// 올바른 방법: 매 tick마다 Date.now()로 남은 시간 계산
function computeRemaining(startedAtMs: number, plannedDurationSec: number): number {
  const elapsedMs = Date.now() - startedAtMs;
  const elapsedSec = Math.floor(elapsedMs / 1000);
  return Math.max(0, plannedDurationSec - elapsedSec);
}

// ❌ 절대 하지 말 것: interval++로 카운트-업/카운트-다운
// let seconds = 1500; setInterval(() => seconds--, 1000); ← 이것은 drift 발생
```

**이유**: 앱이 백그라운드에 있다가 복귀해도 `Date.now()`는 실제 시간을 반영하므로 정확한 남은 시간이 즉시 표시된다 (NFR4).

### ⚠️ Zustand sessionStore 설계 가이드

```typescript
// src/features/session/state/sessionStore.ts
import { create } from 'zustand';
import type { Session } from '../../../domain/sessions/session';

type SessionPhase = 'idle' | 'running' | 'completed' | 'interrupted';

interface SessionState {
  activeSession: Session | null;
  sessionPhase: SessionPhase;
  selectedTopicId: string | null;
  selectedTopicName: string | null;
  // 액션
  startSession: (session: Session) => void;
  endSession: () => void;
  interruptCurrentSession: () => void;
  setSelectedTopic: (id: string | null, name: string | null) => void;
  reset: () => void;
}
```

**주의**:
- Zustand store는 **active UI/session state만** 보관 — Session 엔티티 전체를 복제하되 persisted truth는 SQLite
- `activeSession`에 DB에서 받은 `Session` 객체를 그대로 저장 — 타이머 계산에 `startedAtMs`, `plannedDurationSec` 필요
- `sessionPhase`는 UI 흐름 제어용 — `idle` → `running` → `completed/interrupted` → `idle`

### ⚠️ session-service.ts 구현 가이드

```typescript
// src/features/session/session-service.ts
import { createSession } from '../../domain/sessions/session-repository';
import type { CreateSessionInput } from '../../domain/sessions/session';
import type { Result } from '../../shared/lib/result';
import type { Session } from '../../domain/sessions/session';

export const DEFAULT_STUDY_DURATION_SEC = 25 * 60; // 1500초

export async function beginStudySession(
  topicId: string,
  plannedDurationSec: number = DEFAULT_STUDY_DURATION_SEC,
): Promise<Result<Session>> {
  const input: CreateSessionInput = {
    topicId,
    phaseType: 'study',
    plannedDurationSec,

  return createSession(input);
}
```

### Anti-Patterns to Avoid

- ❌ `setInterval(() => count--, 1000)` 방식 카운트다운 — timestamp 기반만 사용
- ❌ `loading.svg`를 데이터 로딩 상태에 사용 — 세션 진행 중 전용
- ❌ 세션 페이지에서 MainNavigation 표시 — FocusLayout 사용
- ❌ SessionPage에서 직접 SQL 호출 — session-service 경유만
- ❌ sessionStore에 모든 과거 세션 기록 저장 — 현재 active session만
- ❌ `session-repository.ts` 수정 — Story 3.1에서 완성됨, 그대로 사용
- ❌ `topic-repository.ts`, `topic-service.ts` 수정 — 기존 코드 그대로 사용
- ❌ 세션 완료 처리 (Session Outcome Panel) — 이 스토리는 타이머와 집중 모드만, 완료 UI는 Story 3.3
- ❌ 휴식 세션 타이머 — Story 3.4 범위
- ❌ 중단 후 회복 플로우 UI — Story 3.5 범위
- ❌ Tailwind, styled-components, inline styles — CSS Modules + CSS variables만
- ❌ `ProgressBar` 같은 shared UI 신규 생성 — 이 스토리에서는 SessionFocusTimer 내부에 자체 진행률 표시 (shared 분리는 필요 시 향후)
- ❌ macOS 알림 — Story 3.3 범위
- ❌ 통계 갱신 — Epic 5 범위
- ❌ `features/dashboard/` 파일 생성 — Epic 5 범위

### ⚠️ 세션 완료 시점 처리 범위 명확화

이 스토리에서 `useSessionClock`의 `onComplete` 콜백이 호출되면:
1. `completeSession(sessionId)` 호출하여 DB에 기록 (session-repository 사용)
2. sessionStore의 `sessionPhase`를 `'completed'`로 전환
3. **단, Session Outcome Panel(결과 화면)은 Story 3.3에서 구현**
4. 이 스토리에서 세션 완료 시에는 간단한 "세션이 완료되었습니다" 텍스트 + 홈으로 돌아가기 버튼만 표시

마찬가지로 중단 시:
1. `interruptSession(sessionId)` 호출하여 DB에 기록
2. sessionStore의 `sessionPhase`를 `'interrupted'`로 전환
3. "세션이 중단되었습니다" + 홈으로 돌아가기 버튼만 표시 (회복 플로우는 Story 3.5)
