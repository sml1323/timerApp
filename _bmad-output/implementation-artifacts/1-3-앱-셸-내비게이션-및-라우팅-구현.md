## Story

As a 학습자,
I want 메인 화면, 주제 관리 화면, 통계 화면 사이를 자유롭게 이동할 수 있기를,
so that 학습 실행 전, 실행 중, 실행 후 흐름을 끊김 없이 이어갈 수 있다.

## Acceptance Criteria

1. **Given** 앱이 실행되었을 때 **When** 메인 내비게이션을 사용하면 **Then** 홈(`/`), 주제 관리(`/topics`), 통계(`/stats`) 화면으로 이동할 수 있다 (FR28)
2. **And** 현재 위치가 내비게이션에서 시각적으로 구분된다
3. **And** 현재 위치가 ARIA 상태(`aria-current="page"`)로도 구분된다
4. **And** 존재하지 않는 경로 접근 시 NotFound 화면이 표시된다 (FR31)
5. **And** 앱 셸(AppShell)이 모든 라우트에 공통 레이아웃을 제공한다
6. **And** 로그인이나 계정 생성 없이 바로 앱을 사용할 수 있다 (FR39)
7. **And** 인터넷 연결 없이도 앱이 정상 실행된다 (FR37)
8. **And** Wide Desktop / Standard Desktop / Compact Window에서 레이아웃이 적절히 대응한다 (UX-DR10)

### MainNavigation 설계 가이드

```typescript
// src/app/layout/MainNavigation.tsx
import { NavLink } from "react-router";
import styles from "./MainNavigation.module.css";

const NAV_ITEMS = [
  { to: "/", label: "홈", icon: "🏠" },
  { to: "/topics", label: "주제", icon: "📚" },
  { to: "/stats", label: "통계", icon: "📊" },
] as const;

export function MainNavigation() {
  return (
    <nav className={styles.nav} aria-label="메인 내비게이션">
      <ul className={styles.list}>
        {NAV_ITEMS.map(({ to, label, icon }) => (
          <li key={to}>
            <NavLink
              to={to}
              end={to === "/"}
              className={({ isActive }) =>
                `${styles.link} ${isActive ? styles.active : ""}`
              }
            >
              <span className={styles.icon} aria-hidden="true">{icon}</span>
              <span className={styles.label}>{label}</span>
            </NavLink>
          </li>
        ))}
      </ul>
    </nav>
  );
}
```

> **중요:** React Router 7.x `NavLink`는 활성 상태일 때 자동으로 `aria-current="page"`를 설정한다. 별도 수동 처리 불필요.

> **중요:** 홈 라우트(`/`)의 `NavLink`에는 `end` prop을 반드시 추가해야 한다. 그렇지 않으면 `/topics`, `/stats`에서도 홈이 활성으로 표시된다.

### 반응형 전략

UX 문서 기준 3단계 전략:

```css
/* AppShell.module.css */

/* 기본: Standard Desktop (800–1199px) */
.shell {
  display: flex;
  min-height: 100vh;
}
.content {
  flex: 1;
  padding: var(--spacing-6);
  overflow-y: auto;
}

/* Wide Desktop */
@media (min-width: 1200px) {
  /* 사이드바 넓게, 콘텐츠 여유롭게 */
}

/* Compact Window */
@media (max-width: 799px) {
  .shell {
    flex-direction: column;
  }
  /* 사이드바 → 하단 탭 바로 전환 */
}
```

**핵심 원칙:** 어떤 창 크기에서도 `상태 확인 → 주제 선택 → 세션 시작` 핵심 루프가 깨지지 않아야 한다.

### 라우터 통합 변경사항

**현재 상태 (Story 1.1/1.2 결과):**
```typescript
// router.tsx — 현재
export const router = createBrowserRouter([
  {
    path: "/",
    element: <App />,      // App은 단순히 <Outlet /> 렌더
    children: [...]
  }
]);
```

**변경 후:**
```typescript
// router.tsx — Story 1.3 후
import { AppShell } from "./layout";

export const router = createBrowserRouter([
  {
    path: "/",
    element: <AppShell />,  // AppShell이 레이아웃 + Outlet
    children: [
      { index: true, element: <HomeRoute /> },
      { path: "topics", element: <TopicsRoute /> },
      { path: "stats", element: <StatsRoute /> },
      { path: "*", element: <NotFoundRoute /> },
    ],
  },
]);
```

**App.tsx 처리:** AppShell이 루트 레이아웃을 대체하므로 `App.tsx`에서 `<Outlet />`을 렌더하던 역할은 불필요해진다. `App.tsx`를 제거하거나, 비어있게 유지하되 import를 정리한다.

### CSS 변수 활용 원칙

모든 스타일 값은 `tokens.css`의 CSS 변수를 참조해야 한다:
- 색상: `var(--color-*)`, `var(--color-bg-*)`, `var(--color-text-*)`, `var(--color-border-*)`
- 간격: `var(--spacing-*)`
- 글꼴: `var(--font-size-*)`, `var(--font-weight-*)`
- 둥근 모서리: `var(--radius-*)`
- 전환: `var(--transition-*)`
- 그림자: `var(--shadow-*)`
- 포커스: `var(--focus-ring-*)`

❌ 하드코딩 금지: `color: #333`, `padding: 16px`, `font-size: 14px` 등 직접 값 사용 금지

### Anti-Patterns to Avoid

- ❌ Tailwind, styled-components, emotion 등 CSS-in-JS 사용
- ❌ 하드코딩 색상/spacing/font-size (CSS 변수 참조 필수)
- ❌ `outline: none` — 포커스 표시 제거 금지
- ❌ `<a href>` 대신 React Router `<NavLink>` 사용할 것
- ❌ `<BrowserRouter>` 래퍼 패턴 사용 금지 (`createBrowserRouter` 유지)
- ❌ `App.tsx`에서 AppShell과 Outlet을 중복 렌더
- ❌ 전역 CSS 파일에 컴포넌트 스타일 작성 (CSS Modules 사용)
- ❌ FocusLayout 구현 (Story 3.2 범위)
- ❌ 네비 아이콘에 외부 아이콘 라이브러리 도입 (이모지 또는 인라인 SVG로 충분)
